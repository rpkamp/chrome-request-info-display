Example: 
[
    {
        "domain": "^www\\.your-production-website\\.com$",
        "css": "background-color: #f00;",
        "text": "PROD: %ip% | %statusLine% | %header:via%"
    },
    {
        "domain": "^www\\.your-staging-website\\.com$",
        "css": "background-color: #00f;",
        "text": "STAGING: %ip% | %statusLine%"
    },
    {
        "domain": ".*",
        "text": "%ip%"
    }
]

This will:

  • display a red information block with a text like "PROD: 1.2.3.4 | HTTP/1.1 200 OK | 1.1 varnish" on your production site
  • display a blue information block with a text like "STAGING: 1.2.3.4 | HTTP/1.1 200 OK" on your staging site
  • display a gray (the default color) block with text like "1.2.3.4" on any other website

Rules:

  • "domain" is required and is a regular expression (case insensitive by default). Make sure to escape . twice (\\.) because otherwise the JSON parser fails to load your configuration
  • "css" is optional and can contain any CSS you like to be applied to the block
  • "text" is required and can contain placeholders like %ip%. A full list of supported placeholders is listed below. You can use HTML here.
  • Configurations will be tried one by one, the first one that matches is displayed, and all following configurations will be ignored. So be sure to put the most specific "domain" checks first
Placeholder Replacement
%ip% The IP the website is server from
%statusCode% The HTTP status code of the request (e.g. 200)
%statusLine% The HTTP status line of the request (e.g. HTTP/1.1 200 OK)
%url% The fetched URL
%method% The HTTP method used for the request (e.g. GET, POST)
%initiator% What initiated the request
%header:foo% Contents of HTTP header 'foo'. Header name must be lower case, i.e., to get the contents of the "Server" header you need to use %header:server%.

There is some more Chrome internal stuff that can be replaced as well, like %tabId%. For a full list of options see the details object on https://developer.chrome.com/extensions/webRequest#event-onResponseStarted.

Any placeholder that cannot be resolved for a request, like a header that does not exist, will be replaced with undefined
